java之Applet

1.简介

1.1 Applet

什么是 Java Applet？

Java Applet 是一种用 Java 编写的小程序（应用程序小程序)，通常用于在支持 Java 的网页中嵌入动态内容。Applet 通过 <applet> 标签或 <object> 标签嵌入到 HTML 页面中，当用户访问包含 Applet 的网页时，浏览器会下载并执行该 Applet

Java Applet 的工作原理

  1. 嵌入网页：开发者在 HTML 页面中使用 <applet> 或 <object> 标签嵌入 Applet
  2. 下载 Applet：当用户访问该网页时，浏览器会从服务器下载 Applet 的 .class 文件
  3. 执行 Applet：浏览器通过 Java 插件加载并执行 Applet
  4. 与网页交互：Applet 可以与网页中的其他元素（如 JavaScript）进行交互，提供动态和交互性的功能

1.2 GUI

GUI（Graphical User Interface，图形用户界面） 是指通过图形化的元素（如窗口、按钮、图标、菜单、文本框等）来与用户进行交互的界面。相对于早期的命令行界面（CLI，Command Line Interface），GUI 提供了更直观、易用和友好的用户体验，使得用户无需记忆复杂的命令语法，只需通过鼠标点击或触摸屏幕即可操作计算机

GUI 的主要特点

  1. 可视化元素：使用图形化的组件，如按钮、文本框、列表框、图标等，使用户能够直观地理解和操作
  2. 用户友好：界面设计注重用户体验，操作简便，适合各种技术水平的用户
  3. 交互性强：支持鼠标、键盘、触摸等多种输入方式，提供即时反馈，增强用户互动
  4. 多窗口支持：允许同时打开多个窗口，方便用户在不同任务之间切换和管理

GUI 的组成部分

  1. 窗口（Window）：应用程序的主界面区域，可以包含其他组件
  2. 按钮（Button）：用户点击以执行特定操作的控件
  3. 文本框（Textbox）：允许用户输入和编辑文本信息
  4. 标签（Label）：用于显示静态文本，提供信息说明
  5. 列表框（Listbox）和组合框（Combobox）：用于展示选项供用户选择
  6. 菜单（Menu）和工具栏（Toolbar）：提供访问各种功能和命令的快捷方式
  7. 图标（Icon）：代表应用程序、文件或其他对象的图形符号
  8. 对话框（Dialog Box）：用于显示消息、警告或获取用户输入的弹出窗口

1.3 Application

Application（应用程序）是指为了完成特定任务或提供特定功能而设计的软件程序。应用程序通常是为用户在特定环境下（如桌面、移动设备、网页浏览器等）执行特定操作而开发的。与系统软件（如操作系统）不同，应用程序直接面向用户，提供各种实用功能和服务

应用程序的主要特点

  1. 用户导向：应用程序旨在满足用户的具体需求，如办公、娱乐、教育、通信等
  2. 功能专一：每个应用程序通常专注于完成某一类任务，例如文字处理、图像编辑、视频播放等
  3. 交互性强：应用程序通常具有友好的用户界面，允许用户通过图形界面或命令行与程序进行交互
  4. 独立性：许多应用程序可以独立运行，不依赖于其他软件（当然，有些应用可能需要特定的系统环境或依赖库）

应用程序的分类

  1. 桌面应用程序
  • 定义：在个人计算机（如Windows、macOS、Linux）上运行的应用程序
  • 示例
    • 办公软件：Microsoft Office、WPS Office
    • 图像编辑：Adobe Photoshop、GIMP
    • 浏览器：Google Chrome、Mozilla Firefox
  1. 移动应用程序
  • 定义：在移动设备（如智能手机、平板电脑）上运行的应用程序，通常通过应用商店下载安装
  • 示例
    • 社交应用：微信、Facebook
    • 游戏：王者荣耀、Candy Crush
    • 工具类应用：地图导航（如高德地图）、健康管理（如Keep）
  1. 网页应用程序
  • 定义：通过网页浏览器访问和使用的应用程序，无需安装，通常基于Web技术（HTML、CSS、JavaScript）开发
  • 示例
    • 在线办公：Google Docs、Microsoft Office Online
    • 社交媒体：Twitter、LinkedIn
    • 电子商务：淘宝、Amazon
  1. 服务器端应用程序
  • 定义：在服务器上运行的应用程序，提供数据存储、处理和管理服务，常用于网站后台、数据库管理、云计算等
  • 示例
    • Web服务器：Apache HTTP Server、Nginx
    • 数据库管理系统：MySQL、PostgreSQL
    • 云服务平台：AWS、Microsoft Azure
  1. 嵌入式应用程序
  • 定义：嵌入在硬件设备中的应用程序，通常用于控制和管理设备的功能
  • 示例
    • 智能家居设备：智能恒温器（如Nest）、智能音箱（如Amazon Echo）
    • 车载系统：导航系统、娱乐系统
    • 工业控制系统：PLC（可编程逻辑控制器）软件

1.4 Applet与GUI区别

方面 Applet GUI
定义 嵌入在网页中的小程序，用于增加网页的交互性 用户与程序交互的图形界面，独立于浏览器运行
运行环境 需要在支持 Java 的浏览器中运行，依赖浏览器插件 独立运行在操作系统上，通过 Java 虚拟机（JVM）执行
启动方式 由浏览器加载和启动，通常通过 <applet> 标签嵌入网页 通过命令行、双击 JAR 文件或在 IDE 中运行主类启动
生命周期 有特定的生命周期方法（init()、start()、stop()、destroy()） 没有固定的生命周期，依据应用程序的需求管理
安全性 运行在沙箱环境中，安全性受限，需通过数字签名等方式提升权限 安全性依赖于程序的设计和实现，通常拥有更高的权限
用途 主要用于网页中的动态内容和交互，如在线游戏、动画等 创建桌面应用程序的用户界面，如文本编辑器、图像处理软件等
依赖库 基于 Java 的 AWT 或 Swing 库，但主要用于网页嵌入 使用 AWT、Swing、JavaFX 等库构建独立的应用程序界面

1.5 Applet与Application区别

  1. 运行环境
  • Applet
    • 运行平台：主要用于在支持 Java 的网页浏览器中运行
    • 依赖环境：依赖于浏览器提供的 Java 插件，通过浏览器加载和执行
  • Application
    • 运行平台：作为独立的程序在操作系统上运行，不依赖于浏览器
    • 依赖环境：只需要安装 Java 运行环境（JRE）即可运行
  1. 启动方式
  • Applet
    • 由浏览器通过 <applet> 标签或 <object> 标签嵌入网页中启动
    • 浏览器加载网页时，自动加载并执行 Applet
  • Application
    • 通过命令行（如 java 命令）或集成开发环境（IDE）直接运行主类（包含 main 方法的类）启动
  1. 生命周期管理
  • Applet
    • 具有特定的生命周期方法，包括 init()、start()、stop() 和 destroy()，用于管理 Applet 的初始化、启动、暂停和销毁
  • Application
    • 通常只有一个入口点 main 方法，程序的生命周期由开发者自行管理，没有固定的生命周期方法
  1. 用户界面
  • Applet
    • 主要通过图形用户界面（GUI）与用户交互，通常嵌入在网页中，显示在网页的特定区域
    • 使用 AWT 或 Swing 等 GUI 组件库创建界面
  • Application
    • 可以有各种形式的用户界面，包括命令行界面（CLI）、图形用户界面（GUI）等
    • GUI 应用程序同样使用 AWT、Swing、JavaFX 等库创建界面
  1. 安全性
  • Applet
    • 运行在浏览器的沙箱环境中，受到严格的安全限制，无法访问本地文件系统或执行某些敏感操作，除非用户授予特定权限（通过数字签名等方式）
  • Application
    • 作为独立的程序运行，拥有更高的权限，可以访问本地资源、文件系统和其他系统功能，安全性依赖于程序的设计和实现
  1. 交互性
  • Applet
    • 与网页内容紧密结合，可以与 JavaScript 等网页脚本进行交互，增强网页的动态性和互动性
  • Application
    • 通常独立于浏览器，交互性主要体现在程序自身的用户界面和功能上，不直接与网页内容交互
  1. 部署方式
  • Applet
    • 通过将 .class 文件嵌入网页中，用户访问网页时自动下载并执行 Applet
    • 可以使用 JAR 文件打包多个类文件和相关资源，提高加载效率
  • Application
    • 通过分发可执行的 JAR 文件或安装包，用户自行下载并运行程序
    • 可以包含独立的启动脚本或安装程序，方便用户安装和使用

2.生命周期

使用生命周期方法管理 Applet 行为

  • init()：在 Applet 被加载时调用，用于初始化 Applet
  • start()：在 Applet 启动时调用，通常在 init() 之后或用户返回包含 Applet 的页面时调用
  • stop()：在 Applet 停止时调用，通常在用户离开包含 Applet 的页面时调用
  • destroy()：在 Applet 被卸载时调用，用于释放资源

Applet（应用程序小程序）的生命周期指的是从 Applet 被加载到浏览器中开始，到最终被卸载的整个过程。理解 Applet 的生命周期对于正确地初始化资源、处理用户交互以及释放资源非常重要。Applet 的生命周期主要由以下几个阶段组成，每个阶段都对应着特定的方法

  1. 加载（Loading）
  • 当浏览器加载包含 Applet 的网页时，会加载 Applet 类。这一过程通常由浏览器自动完成，开发者无需干预
  1. 初始化（Initialization）
  • 方法: init()
  • 描述: 这是 Applet 的第一个生命周期方法。在 Applet 被加载后，浏览器会调用 init() 方法。在这里，开发者可以进行一次性的初始化操作，如设置初始变量、加载资源、初始化界面组件等
  • 调用次数: 仅调用一次
  1. 启动（Start）
  • 方法: start()
  • 描述: 在 init() 方法执行完毕后，或者当用户从其他页面返回当前包含 Applet 的页面时，浏览器会调用 start() 方法
  • 调用次数: 每次 Applet 变得可见时调用，可能多次调用
  1. 绘制（Painting）
  • 方法: paint(Graphics g)
  • 描述: paint() 方法用于绘制 Applet 的图形界面。每当 Applet 需要重新绘制时（如窗口被覆盖后重新显示），浏览器会调用此方法
  • 调用次数: 根据需要多次调用
  1. 停止（Stop）
  • 方法: stop()
  • 描述: 当用户离开包含 Applet 的页面，或者浏览器窗口被最小化时，浏览器会调用 stop() 方法
  • 调用次数: 每次 Applet 变得不可见时调用，可能多次调用
  1. 销毁（Destruction）
  • 方法: destroy()
  • 描述: 当浏览器关闭或者 Applet 被卸载时，浏览器会调用 destroy() 方法
  • 调用次数: 仅调用一次

生命周期流程图

  1. 加载 → 初始化 (init()) → 启动 (start()) → 绘制 (paint())
  2. 当 Applet 不可见时：停止 (stop())
  3. 当 Applet 再次可见时：启动 (start()) → 绘制 (paint())
  4. 当 Applet 被卸载或浏览器关闭时：销毁 (destroy())

3.参数获取与处理

  • 在 Java Applet 中，参数的获取与处理是通过 HTML 标签中的 <param> 元素来实现的

  • 在 HTML 中嵌入 Applet 时，可以使用 <param> 标签为 Applet 传递参数。每个 <param> 标签包含两个属性：

    • name：参数的名称
    • value：参数的值

  • Java Applet 提供了 getParameter(String name) 方法用于获取传递的参数值。该方法返回指定名称的参数值，如果未找到则返回 null

4.事件处理

  • Java Applet 使用 AWT（Abstract Window Toolkit） 事件模型来处理事件。该模型基于以下几个核心概念：
    • 事件源（Event Source）：产生事件的组件，如按钮、窗口、键盘等
    • 事件对象（Event Object）：描述事件的详细信息，如事件类型、发生位置等
    • 事件监听器（Event Listener）：负责接收并处理事件的对象，必须实现相应的监听器接口

常见的 Java Applet 事件类型

  1. 动作事件（Action Events）
  2. 鼠标事件（Mouse Events）
  3. 键盘事件（Keyboard Events）
  4. 窗口事件（Window Events）
  5. 组件事件（Component Events）
  6. 焦点事件（Focus Events）
  7. 输入法事件（Input Method Events）

4.1 动作事件（Action Events）

  • 描述：动作事件通常与用户的操作相关，如按钮点击、菜单选择等

  • 监听器接口：ActionListener

  • 常用方法：

    • actionPerformed(ActionEvent e)

4.2 鼠标事件（Mouse Events）

  • 描述：鼠标事件涉及鼠标的各种操作，如点击、进入、离开、拖动等
  • 监听器接口：MouseListener 和 MouseMotionListener
  • 常用方法：
    • MouseListener
      • mouseClicked(MouseEvent e)
        • 描述：当用户点击（按下并释放）鼠标按钮时触发
        • 常用场景：计数点击次数、选择项目等
      • mousePressed(MouseEvent e)
        • 描述：当用户按下鼠标按钮时触发
        • 常用场景：开始拖拽操作、游戏中的动作触发等
      • mouseReleased(MouseEvent e)
        • 描述：当用户释放鼠标按钮时触发
        • 常用场景：结束拖拽操作、确认选择等
      • mouseEntered(MouseEvent e)
        • 描述：当鼠标指针进入组件区域时触发
        • 常用场景：显示提示信息、高亮显示等
      • mouseExited(MouseEvent e)
        • 描述：当鼠标指针离开组件区域时触发
        • 常用场景：隐藏提示信息、取消高亮显示等
    • MouseMotionListener
      • mouseDragged(MouseEvent e)
        • 描述：当用户按下鼠标按钮并移动鼠标时触发
        • 常用场景：绘图应用中的画图、拖拽操作等
      • mouseMoved(MouseEvent e)
        • 描述：当用户移动鼠标指针时触发
        • 常用场景：实现光标跟随、实时绘图等

4.3 键盘事件（Keyboard Events）

  • 描述：键盘事件涉及用户的键盘输入，如按键按下、释放、键入等

  • 监听器接口：Listener

  • 常用方法：

    • keyTyped(KeyEvent e)
      • 描述：当用户键入一个字符时触发。例如，按下并释放一个字母或数字键
      • 常用场景：处理文本输入、字符验证等
    • keyPressed(KeyEvent e)
      • 描述：当用户按下任意键时触发，包括功能键、控制键等
      • 常用场景：游戏中的按键控制、快捷键处理等
    • keyReleased(KeyEvent e)
      • 描述：当用户释放之前按下的键时触发
      • 常用场景：检测按键释放以触发特定动作，如射击游戏中的开火

  • 为了确保 Applet 能够接收到键盘事件，必须确保它具有焦点。可以通过以下方法实现：

    • **setFocusable(true)**：使 Applet 可以获得焦点
    • **requestFocus()**：请求焦点

4.4 窗口事件（Window Events）

  • 描述：窗口事件涉及窗口的操作，如打开、关闭、最小化、调整大小等

  • 监听器接口：WindowListener

  • 常用方法：

    • windowOpened(WindowEvent e)
    • windowClosing(WindowEvent e)
    • windowClosed(WindowEvent e)
    • windowIconified(WindowEvent e)
    • windowDeiconified(WindowEvent e)
    • windowActivated(WindowEvent e)
    • windowDeactivated(WindowEvent e)

  • 注意：在 Applet 中直接处理窗口事件较少见，通常由浏览器或容器管理

4.5 组件事件（Component Events）

  • 描述：组件事件涉及组件的大小、位置变化等

  • 监听器接口：ComponentListener

  • 常用方法：

    • componentResized(ComponentEvent e)
      • 触发条件：当组件的大小被调整时触发，例如用户拖动窗口边框改变大小
      • 常用属性：
        • e.getComponent()：获取发生事件的组件
        • getWidth() 和 getHeight()：获取组件的新宽度和高度
      • 示例用途：
        • 动态调整布局以适应新的组件大小
        • 重新绘制图形或图像以匹配新的尺寸
    • componentMoved(ComponentEvent e)
      • 触发条件：当组件的位置发生变化时触发，例如窗口被拖动到新的位置
      • 常用属性：
        • e.getComponent()：获取发生事件的组件
        • getX() 和 getY()：获取组件的新位置坐标
      • 示例用途：
        • 跟踪组件的位置以进行特定的交互，如拖拽操作
        • 更新界面元素的位置信息
    • componentShown(ComponentEvent e)
      • 触发条件：当组件被显示时触发，例如窗口从隐藏状态变为可见
      • 常用属性：
        • e.getComponent()：获取发生事件的组件
      • 示例用途：
        • 初始化显示内容，如加载数据或启动动画
        • 启动定时器或后台任务
    • componentHidden(ComponentEvent e)
      • 触发条件：当组件被隐藏时触发，例如窗口被最小化或覆盖
      • 常用属性：
        • e.getComponent()：获取发生事件的组件
      • 示例用途：
        • 暂停动画或后台任务以节省资源
        • 保存当前状态或数据

4.6 焦点事件（Focus Events）

  • 描述：焦点事件涉及组件获得或失去输入焦点

  • 监听器接口：FocusListener

  • 常用方法：

    • focusGained(FocusEvent e)
      • 触发条件：当组件获得输入焦点时触发，例如用户点击文本框或使用 Tab 键导航到该组件
      • 常用属性：
        • e.getComponent()：获取获得焦点的组件
        • e.isTemporary()：判断焦点变化是否为临时性的（例如弹出菜单时）
      • 示例用途：
        • 显示输入提示或光标。
        • 启用特定的输入验证或自动完成功能
        • 改变组件的视觉样式以指示当前焦点
    • focusLost(FocusEvent e)
      • 触发条件：当组件失去输入焦点时触发，例如用户点击其他组件或按下 Tab 键导航到其他组件
      • 常用属性：
        • e.getComponent()：获取失去焦点的组件
        • e.isTemporary()：判断焦点变化是否为临时性的
      • 示例用途：
        • 验证用户输入的数据是否正确
        • 保存用户输入的数据
        • 恢复组件的默认视觉样式

4.7 输入法事件（Input Method Events）

描述：输入法事件涉及输入法相关的操作，如输入法文本转换

监听器接口：InputMethodListener

常用方法：

  • inputMethodTextChanged(InputMethodEvent e)
    • 触发条件：当输入法文本发生变化时触发，例如用户在输入过程中选择了不同的候选词或完成了文本转换
    • 常用属性：
      • e.getText()：获取输入法提供的文本内容
      • e.getCommittedText()：获取已提交的文本内容
      • e.getInsertedText()：获取插入的文本内容
      • e.getRemovedText()：获取被移除的文本内容
    • 示例用途：
      • 实时显示和处理用户的输入过程，如候选词选择
      • 记录用户的输入历史或进行实时校验
  • caretPositionChanged(InputMethodEvent e)
    • 触发条件：当插入符号（光标）的位置在输入过程中发生变化时触发
    • 常用属性：
      • e.getCaret()：获取当前的插入符号（光标）对象
      • e.getDot()：获取插入符号的位置
      • e.getMark()：获取标记位置（通常与选择范围相关）
    • 示例用途：
      • 更新界面以反映当前的插入符号位置，如实时显示光标位置
      • 实现与插入符号位置相关的动态效果，如跟随光标的提示信息

4.8 事件适配器类

Java 提供了多种 适配器类（Adapter Classes），这些类实现了监听器接口，并为所有方法提供了空实现。可以通过继承适配器类并重写需要的方法，减少需要实现的方法数量，从而简化代码

常用适配器类：

  • MouseAdapter
  • KeyAdapter
  • ComponentAdapter
  • FocusAdapter
