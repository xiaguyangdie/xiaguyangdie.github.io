2.1.2 元素定位

优先使用 ID > Name > CSS Selector > XPath

定位方式 速度 稳定性 适用场景
ID 最快 高 元素有唯一 id 时
Name 较快 中 表单字段
Class Name 中 低 元素有唯一样式类
Tag Name 慢 低 定位一组同类标签
Link Text 快 高 超链接文本明确
Partial Link Text 快 中 超链接文本部分已知
XPath 慢 中 复杂动态元素
CSS Selector 较快 高 现代 Web 应用，优先推荐

2.1.4 元素交互

  • 文本输入：sendKeys()
  • 点击操作：click()

2.1.5 等待机制

  • 显式等待：WebDriverWait + ExpectedConditions（推荐）
  • 隐式等待：driver.manage().timeouts().implicitlyWait()
  • 强制等待：Thread.sleep()（不推荐）

6.补充

6.1 Selenium 版本演进

Selenium 1.x（已淘汰）

  • 核心组件：Selenium RC（Remote Control）
  • 特点：
    • 通过 HTTP 协议直接与浏览器交互
    • 需要安装 Selenium RC 服务器
    • 存在安全性和稳定性问题
  • 现状：官方已不再维护，不推荐使用

Selenium 2.x（过渡版本）

  • 核心组件：WebDriver + Selenium RC
  • 特点：
    • 引入 WebDriver API（面向对象，更稳定）
    • 支持多浏览器、多语言绑定
    • Selenium RC 仍兼容，但官方推荐使用 WebDriver
  • 现状：逐渐被 Selenium 3/4 取代

Selenium 3.x

  • 核心组件：WebDriver（标准化）、Selenium Grid、IDE
  • 关键变化：
    • WebDriver 成为唯一官方支持的 API（Selenium RC 彻底废弃）
    • WebDriver 遵循 W3C 标准（跨浏览器兼容性提升）
    • GeckoDriver（Firefox 驱动）成为独立组件（之前内置于 Firefox）
  • 支持浏览器：Chrome、Firefox、Edge、Safari 等

Selenium 4.x（最新版本）

  • 核心组件：WebDriver 4.x、Selenium Grid 4.x、IDE 插件化
  • 新增功能：
    • 相对定位器（Relative Locators）：通过元素之间的相对位置定位（如 above(), toRightOf()）。
    • 改进的 Grid 架构：支持 Docker 部署、动态节点管理
    • DevTools API 集成：直接调用浏览器原生 DevTools 协议（如 Chrome DevTools）
    • W3C 协议增强：支持更多浏览器操作（如截图、网络拦截）
  • 兼容性：完全遵循 W3C WebDriver 标准

6.2 Selenium 的核心组成

Selenium WebDriver

  • 作用：通过浏览器驱动控制浏览器行为（核心组件）
  • 支持的浏览器：
    • Chrome（ChromeDriver）
    • Firefox（GeckoDriver）
    • Edge（EdgeDriver）
    • Safari（SafariDriver）
    • Opera（OperaDriver）
  • 支持的语言：Java、Python、C#、Ruby、JavaScript 等
  • 核心 API：
    • 浏览器操作：get(), quit(), refresh()
    • 元素交互：findElement(By), click(), sendKeys()
    • 等待机制：显式等待（WebDriverWait）、隐式等待（implicitlyWait）

Selenium IDE（集成开发环境）

  • 作用：浏览器插件，用于录制和回放测试脚本（适合快速原型开发）
  • 特点：
    • 支持 Firefox 和 Chrome
    • 生成基础测试脚本（可导出为多种语言代码）
  • 局限性：仅适用于简单场景，复杂逻辑需手动完善

Selenium Grid

  • 作用：分布式测试工具，支持并行执行测试用例
  • 架构：
    • Hub：中央节点，管理测试任务分发
    • Nodes：实际执行测试的浏览器实例
  • 功能：
    • 跨浏览器、跨平台测试
    • 动态扩展测试节点（如通过 Docker 部署）

Selenium DevTools（浏览器开发者工具集成）

  • 作用：直接调用浏览器 DevTools 协议（如 Chrome DevTools）
  • 功能：
    • 网络监控（请求/响应拦截）
    • 性能分析（CPU、内存占用）
    • 控制浏览器行为（如模拟移动设备、修改 User-Agent）
