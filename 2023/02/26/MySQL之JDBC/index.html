数据库链接–执行完毕–释放十分消耗资源

池化技术：准备一些预先的资源，过来就链接预先准备好的

若常用连接数10个

最小连接数：10个即可

最大连接数：15 业务最高承载上限，超过此值则排队等待

参数配置

  • 基础配置参数（必填）

    参数名‌ ‌说明‌ ‌示例值‌
    driverClassName JDBC驱动类全限定名 com.mysql.jdbc.Driver
    url 数据库连接URL（含协议、主机、端口、数据库名）‌ jdbc:mysql://localhost:3306/mydb
    username 数据库用户名 root
    password 数据库密码 password

  • 连接池容量控制参数

    ‌参数名‌ ‌默认值‌ ‌说明‌
    initialSize 0 初始化连接数，启动时创建的连接数量
    maxTotal 8 最大活跃连接数（旧版称maxActive）
    maxIdle 8 最大空闲连接数 > 超过时多余连接被释放
    minIdle 0 最小空闲连接数 < 低于时创建新连接
    maxWaitMillis -1 (无限) 获取连接最大等待时间（毫秒，超时抛异常）

  • 连接健康检测机制

    ‌参数名‌ ‌默认值‌ ‌说明
    validationQuery 连接有效性检测SQL（必填）
    ✅ MySQL: SELECT 1 ✅ Oracle: SELECT 1 FROM DUAL
    testOnBorrow false 借出连接时是否检测有效性（‌生产推荐开启‌）
    testWhileIdle false 空闲时是否定期检测（需配合timeBetweenEvictionRunsMillis）
    timeBetweenEvictionRunsMillis -1 (关闭) 空闲连接检测周期（毫秒）
    minEvictableIdleTimeMillis 1800000 连接空闲超时时间（默认30分钟），超过则回收
    numTestsPerEvictionRun 3 每次检测的连接数量（建议≥maxIdle）

  • 连接泄漏防护

    参数名‌ ‌默认值‌ ‌‌说明
    removeAbandonedOnBorrow false 是否自动回收泄露连接（未主动关闭的连接）
    removeAbandonedTimeout 300 泄漏连接超时时间（秒）
    logAbandoned false 是否记录泄露连接的堆栈信息（调试用）

  • 事务

    参数名‌ ‌默认值‌ ‌‌说明‌
    defaultAutoCommit true 是否自动提交事务 （true=每条SQL单独提交，false=需手动commit/rollback）
    defaultReadOnly false 连接默认是否为只读模式
    defaultTransactionIsolation 依赖数据库默认值 事务隔离级别 （如READ_COMMITTED、REPEATABLE_READ等）

  • 连接属性（扩展配置）（connectionProperties=useUnicode=true;characterEncoding=UTF-8 ）

    属性键‌ ‌示例值‌ ‌说明‌
    useUnicode true 是否使用Unicode字符集（处理中文等非ASCII字符必需）
    characterEncoding UTF-8 指定字符编码格式（需与数据库编码一致）

参数配置

  • 数据库连接参数（必填）‌

    ‌参数‌ ‌说明‌ ‌示例值‌
    c3p0.driverClass JDBC驱动类全名 com.mysql.jdbc.Driver
    c3p0.jdbcUrl 数据库连接URL jdbc:mysql://host:3306/db_name
    c3p0.user 数据库用户名 root
    c3p0.password 数据库密码 123456

  • 连接池容量控制 ‌

    ‌参数‌ ‌默认值‌ ‌功能‌
    c3p0.initialPoolSize 3 初始连接数
    c3p0.minPoolSize 3 最小保持连接数
    c3p0.maxPoolSize 15 最大连接数
    c3p0.acquireIncrement 3 连接不足时一次性新增连接数
    c3p0.maxIdleTime 0 空闲连接超时时间（秒），0=永不过期

  • 连接健康监测

    参数‌ ‌‌默认值‌‌ ‌说明‌
    c3p0.idleConnectionTestPeriod 0（不检测）‌ 空闲连接测试周期（秒）
    c3p0.preferredTestQuery 测试SQL语句
    c3p0.testConnectionOnCheckin false 归还连接时是否验证有效性

  • 高级容错参数

    参数‌ ‌‌默认值‌‌ ‌说明‌
    c3p0.unreturnedConnectionTimeout 0 未关闭连接强制回收时间（秒）
    c3p0.debugUnreturnedConnectionStackTraces false 启用后会在强制回收时打印连接获取时的堆栈轨迹
    c3p0.checkoutTimeout 0（无限等待） 获取连接超时时间（毫秒）
    c3p0.maxStatements 0（不缓存） 缓存PreparedStatement数量

  • C3P0连接池通过ComboPooledDataSource构造函数自动加载c3p0.properties配置文件，这是由其内部机制实现的，具体原理如下

    • 自动加载机制‌
      C3P0会在类路径下自动查找名为c3p0.properties的配置文件，无需显式指定路径‌。当调用无参构造函数new ComboPooledDataSource()时，会自动解析该文件中的配置项（需以c3p0.为前缀）‌
    • 配置优先级‌
      若同时存在c3p0-config.xml和c3p0.properties，默认优先加载XML配置‌。但无XML时，properties文件会成为唯一配置源‌
    • 底层实现‌
      ComboPooledDataSource初始化时会通过C3P0PooledConnectionPoolManager读取配置，并建立连接池管理逻辑‌。连接参数（如driverClass、jdbcUrl）通过反射动态注入到数据源对象中

Druid

参数配置

‌参数名‌ ‌默认值‌ ‌含义说明‌
‌基础连接配置‌
driverClassName 自动识别 JDBC驱动类名（建议显式配置）
url - 数据库连接地址（不同数据库格式不同）
username - 数据库用户名
password - 数据库密码（支持加密配置）
‌连接池容量控制‌
initialSize 0 初始化时建立的物理连接数（启动时不创建连接）
minIdle - 最小空闲连接数（低于此值会创建新连接）
maxActive 8 最大活跃连接数（连接池最大容量）
maxWait 30000(30秒) 获取连接的最大等待时间（毫秒）
‌连接检测与回收‌
validationQuery - 连接有效性检测SQL（如SELECT 1）
testWhileIdle false 空闲时检测连接有效性（建议开启）
testOnBorrow true 获取连接时检测有效性（影响性能）
testOnReturn false 归还连接时检测有效性（不推荐开启）
timeBetweenEvictionRunsMillis - 空闲连接检测间隔（毫秒）
minEvictableIdleTimeMillis 1800000(30分钟) 连接保持空闲的最短时间（超时关闭）
maxEvictableIdleTimeMillis 25200000(7小时) 连接保持空闲的最长时间（强制关闭）
‌监控与安全‌
filters - 扩展插件：stat（监控）、wall（防火墙）、log4j（日志）
removeAbandoned false 是否回收泄露连接（未关闭的连接）
removeAbandonedTimeout 300 泄露连接回收阈值（秒）
‌高级优化‌
maxPoolPreparedStatementPerConnectionSize - 每个连接缓存的PreparedStatement数
useUnfairLock false 使用非公平锁（提升并发性能）
queryTimeout - SQL查询超时时间（毫秒）
